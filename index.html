重要三件事 — 产品方案

01
背景
痛点1：重要信息被淹没
欠费短信淹没在营销短信中易忽略
上周挂的专家号被后来的短信淹没，差点忘了明天要去
领导给我打了2个电话，我开静音了没听到
痛点2：未来需要关注的事情记不住
5点有个评审会，上次会议遗留待办还未完成
演唱会门票马上就要开始抢票了
黄金涨到了我设的止盈线，再不卖可能回落

产品定义

将「近期被忽略的重要信息」和「即将需要行动的事项」主动呈现给用户

推送时机（第一期：固定三次）

早间 07:30 — 今日预览：汇总今天的日程 + 昨晚遗留的未处理重要信息（欠费/快递/未接来电）
午间 12:00 — 午间提醒：下午需要行动的事项（出发建议/即将截止/上午新增的重要信息）
晚间 20:00 — 今日复盘：未完成事项汇总 + 明日预告（明天第一件事）

静默规则

正常推送：到固定时间点即推送，无论有几条事项
跳过推送：勿扰模式开启时延后到结束后推送；当次无任何事项时不推送空卡片

为什么统一到日视图

所有入口统一落地到日历日视图，而非分散到不同页面。好处：1）用户心智统一——"重要的事都在日历里"；2）日视图作为信息汇聚页，同时展示日程+非日程事项，用户一屏处理完；3）避免新增独立App/页面的学习成本。

06
完整推送策略规则

本章是推送系统的完整规则定义，供开发和测试对齐。

6.1 推送时机
时段 时间 推送内容 设计意图
早间07:30今天的全部日程概览 + 昨晚以来未处理的重要信息（欠费/快递/未接来电）帮用户规划一天
午间12:00下午需要行动的事项（出行建议/即将截止/上午新增的重要信息）帮用户决策下午
晚间20:00未完成事项汇总 + 明日第一件事预告收尾 + 安心入睡
6.2 数据扫描范围

每次推送前触发一次扫描：

往回看

自上一次推送以来所有新增/未处理的信息：短信、未接来电、日历变更、天气预警

往前看

到下一次推送时间点之前需要行动的事项：
• 早间 → 看全天
• 午间 → 看到晚间（20:00）
• 晚间 → 看到明天早间（07:30）

6.3 优先级与选取

每次推送最多展示 3条（"重要三件事"），按优先级从高到低选取：

🔴 高 = 不可逆损失
有多少展示多少（不受3条限制）
航班取消/延误
逾期停机
协议签署截止
🟡 中 = 有时间窗但会误事
高优占完后按紧迫程度填充
还款到期
会议出行建议
就医挂号提醒
🟢 低 = 知道更好但不急
仅高+中不足3条时露出
快递到柜
降价通知
天气变化
超过3条时

通知摘要显示"今天有5件事需关注"，点击进入日视图看完整列表。日视图内不限条数，全部展示。

6.4 推送条件
条件 规则
正常推送到固定时间点、存在 ≥1 条有效事项
跳过推送本时段所有事项过滤后为 0 条（不推空卡片）
延后推送勿扰模式开启中 → 勿扰结束后立即补推一次
用户可配设置页可关闭任一时段；可调整时间（±1小时）；可关闭整个功能
6.5 通知去重
核心判断标准

不是"这个事件出现过没有"，而是——这次出现能让用户多知道什么或多做什么？

场景 是否推 原因
早间提到"下午3点有会" → 午间又提到同一个会✅ 推午间新增了"建议2:15出发"，有增量信息
午间提到"下午3点有会" → 14:45 日历原生弹提醒✅ 推不同时间、不同目的（规划 vs 执行）
14:45 重要三件事弹通知 → 14:46 日历也弹通知❌ 合并同一5分钟窗口同一事件，保留信息量大的那条
早间推了"航班正常" → 9点航班延误了✅ 推状态变化 = 新信息，触发即时推送
早间推了"快递到柜" → 午间用户还没取❌ 不推没有新信息，用户只是还没行动
午间推了"还款今日到期" → 晚间仍未还✅ 推紧迫度升级了（距截止更近），晚间汇总应再提

事项唯一标识规则

{source}_{关键词}_{日期}_{状态}

• 同一 ID + 同一状态，在同一推送时段内只出现一次
• 状态变化 → 生成新 ID → 可以再次推送
• 事项可持续存在于日视图页面（被动展示不受去重限制）

6.6 触达方式
入口 形态 用户场景
锁屏单条摘要通知（🔴高优先级时升级为岛展开）拿起手机第一眼
负一屏独立卡片列表，逐条可操作主动下滑查看
日历月视图AI建议卡（一句话摘要）打开日历时顺带看到

所有入口点击 → 统一跳转 日历日视图改造页（唯一落地页）。

6.7 静默规则
场景 处理
勿扰模式开启中延后到勿扰结束后立即补推
用户正在通话中延后到通话结束后5分钟
用户刚点击了推送（5分钟内）下一时段正常推，不跳过
用户标记了"不相关"同类事项置信度降低，连续3次后该类不再主动推
连续3天推送均无事项降频为仅早间推送
连续7天无互动暂停推送，下次打开日历时恢复
6.8 即时推送（例外规则）

以下场景不等到下一个时间点，直接即时推送：

触发场景 说明
航班/火车取消或延误损失不可逆，1分钟都不能等
同一号码连续3次未接来电可能是紧急联系人
协作日程被他人修改时间可能导致迟到或冲突
即时推送限流

复用日历通知渠道，不新建 channel。每天即时推送上限 3条，防止极端情况下刷屏。超过3条时剩余的并入下一个固定时段推送。
