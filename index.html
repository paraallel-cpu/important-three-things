小爱简报

01
背景
痛点1：早晨信息分散，要分别打开多个 App
起床后要看天气决定穿什么、查日历今天有什么安排、刷新闻了解发生了什么
信息散落在天气、日历、新闻多个入口，缺少一个"一眼看完今天"的地方
痛点2：当天的天气/日程冲突没人提前提醒
上午要下雨却没带伞；下午两个日程时间撞车到出发才发现
这些"今天才生效"的提醒，前一晚记不住、当天又来不及

产品定义

每天早晨由超级小爱主动生成一份专属早报，一屏读完今日要闻与日程：天气提醒 → 今日日程（含冲突预警）→ 待办事项 → AI 感兴趣的资讯。

生成与推送时机

每天早晨固定时段生成（默认 07:30 前后，跟随用户起床/解锁时机微调）
生成后通过锁屏通知、桌面"小爱建议"卡、负一屏多触点同步触达
当日无任何日程/待办时，仅保留天气 + 资讯，不推空卡

与「今日事项」的边界

小爱简报：早间一次性全景播报，含资讯，强调"开启今天"
今日事项：全天动态事项追踪，不含资讯，强调"别漏事"
打通：简报中的日程/待办点击后进入同一日历日视图落地页

03
桌面触点

早报生成后在两个桌面位置同步呈现入口，点击均跳转「日报详情页」落地页。

锁屏：简报通知卡 + 资讯通知
早间解锁第一眼即见，"去查看"直达落地页，下方挂今日资讯通知
桌面：「小爱建议」卡
常驻桌面，摘要式预览今日日程/待办/开票提醒，不点也能扫到关键信息

04
落地页板块排序（日报详情页）

所有桌面触点点击后统一进入「日报详情页」。板块自上而下按"问候 → 天气 → 今日日程 → 待办 → 资讯"排序——越靠上越与"今天必须知道/要做的事"相关，资讯作为延伸阅读垫底。

07
今日事项

把日历的「日视图」从一张静态台历壁纸，改造成用户每天主动来看的「今天该关注什么」信息页，并通过锁屏 / 桌面 / 负一屏多入口主动触达。本章按 现状与痛点 → 新的设计样式 → 从哪里触达 → 推送时机 一次讲清，界面直接看图。

7.1 为什么改：原来的日视图是一张「台历壁纸」
老版日视图主体是一张壁纸 + 一个时间轴，用户看不到"今天到底有几件事、哪件最该紧张"，重要信息还散落在短信、各 App 通知里。
痛点1：重要信息被淹没
欠费短信淹没在营销短信中易忽略
上周挂的专家号被后来的短信淹没，差点忘了明天要去
领导给我打了2个电话，我开静音了没听到
痛点2：未来需要关注的事情记不住
5点有个评审会，上次会议遗留待办还未完成
演唱会门票马上就要开始抢票了
黄金涨到了我设的止盈线，再不卖可能回落

产品定义

展示用户今天所有需要关注的事项，通过 AI 分析优先级，突出紧急 / 重要事项，标记接下来该做的事——一屏看完今天，不必再翻短信和各 App。

7.2 新的设计样式：日视图 = 今日全景信息页
保留台历壁纸入场，但默认态就在壁纸下方露出「需关注」摘要；用户上滑后壁纸收起，整页变成今日全景信息列表：顶部天气 → 需关注事项置顶（红字徽章）→ 今日事项列表（按时间排序、远期灰显）→ 底部 Tab。它同时也是小爱简报里日程板块的二级落地页，两个功能共用同一页面。下面是线上当前样式 → 改版默认态 → 改版上滑态的对比。

7.4 推送时机：定时推 + 只推有增量的事
第一期固定三次推送，配合去重与静默规则，保证"该提醒的不漏、重复的不烦"。

推送时机（第一期：固定三次）

早间 07:30 — 今日全部事项概览 + 优先级标注 + 昨晚新增的未处理信息
午间 12:00 — 下午事项更新（出发建议 / 新增变更 / 即将截止）
晚间 20:00 — 未完成事项汇总 + 明日预告

核心去重原则

判断标准不是"事件出现过没有"，而是 —— 这次出现能让用户多知道什么或多做什么？

有增量才推：状态变化（航班延误）、新增出发建议、紧迫度升级（距截止更近）
无增量合并 / 不推：同事件同状态、用户只是还没行动

静默规则（节选）

已开启小爱简报：跳过早间推送（简报已覆盖），仅保留午间 + 晚间
勿扰 / 通话中：延后到结束后补推；无事项：不推空卡片
长期无互动：连续 3 天无事项降频；连续 7 天无互动暂停，下次打开日历恢复

完整的推送时机 / 数据来源 / 推送条件 / 通知去重 / 触达形态 / 静默规则定义见下文 7.6 完整推送策略规则。
