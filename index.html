今日事项 — 产品方案

01
背景
痛点1：重要信息被淹没
欠费短信淹没在营销短信中易忽略
上周挂的专家号被后来的短信淹没，差点忘了明天要去
领导给我打了2个电话，我开静音了没听到
痛点2：未来需要关注的事情记不住
5点有个评审会，上次会议遗留待办还未完成
演唱会门票马上就要开始抢票了
黄金涨到了我设的止盈线，再不卖可能回落

产品定义

展示用户今天所有需要关注的事项，通过AI分析优先级，突出紧急/重要事项，标记接下来该做的事

推送时机（第一期：固定三次）

早间 07:30 — 今日全部事项概览 + 优先级标注 + 昨晚新增的未处理信息
午间 12:00 — 下午事项更新（出发建议/新增变更/即将截止）
晚间 20:00 — 未完成事项汇总 + 明日预告

静默规则

正常推送：到固定时间点即推送，无论有几条事项
跳过推送：勿扰模式开启时延后到结束后推送；当次无任何事项时不推送空卡片

为什么统一到日视图

所有入口统一落地到日历日视图，而非分散到不同页面。好处：1）用户心智统一——"重要的事都在日历里"；2）日视图作为信息汇聚页，同时展示日程+非日程事项，用户一屏处理完；3）避免新增独立App/页面的学习成本。

07
完整推送策略规则

本章是推送系统的完整规则定义，供开发和测试对齐。

7.1 推送时机
时段 时间 推送内容 设计意图
早间07:30今日全部事项（优先级排序）+ 昨晚新增未处理信息一屏总览今天所有事
午间12:00事项状态更新（新增变更/出行建议/即将截止）刷新下午安排
晚间20:00未完成事项汇总 + 明日预告收尾 + 安心入睡
7.2 数据来源

今日事项的数据聚合自以下来源：

日历本身

用户手动创建的日程、会议邀请、节假日等日历内原生事项

AI智能提取

从短信/订单页中自动识别的日程：航班、火车、酒店、电影、还款、演出、餐厅预订等

7.3 推送条件
条件 规则
正常推送到固定时间点、存在 ≥1 条有效事项
跳过推送本时段所有事项过滤后为 0 条（不推空卡片）
延后推送勿扰模式开启中 → 勿扰结束后立即补推一次
无事项本时段扫描后事项为 0 条，不推送空卡片
7.4 通知去重
核心判断标准

不是"这个事件出现过没有"，而是——这次出现能让用户多知道什么或多做什么？

场景 是否推 原因
早间提到"下午3点有会" → 午间又提到同一个会✅ 推午间新增了"建议2:15出发"，有增量信息
午间提到"下午3点有会" → 14:45 日历原生弹提醒✅ 推不同时间、不同目的（规划 vs 执行）
14:45 今日事项弹通知 → 14:46 日历也弹通知❌ 合并同一5分钟窗口同一事件，保留信息量大的那条
早间推了"航班正常" → 9点航班延误了✅ 推状态变化 = 新信息，午间推送时更新
早间推了"快递到柜" → 午间用户还没取❌ 不推没有新信息，用户只是还没行动
午间推了"还款今日到期" → 晚间仍未还✅ 推紧迫度升级了（距截止更近），晚间汇总应再提

事项唯一标识规则

{source}_{关键词}_{日期}_{状态}

• 同一 ID + 同一状态，在同一推送时段内只出现一次
• 状态变化 → 生成新 ID → 可以再次推送
• 事项可持续存在于日视图页面（被动展示不受去重限制）

7.5 触达方式
入口 形态 用户场景
锁屏胶囊通知（毛玻璃卡片内胶囊标签列表）拿起手机第一眼
桌面小部件344×160 / 160×160 事项列表卡桌面随时可见
负一屏紧凑卡片（标题+红字异常提示）主动下滑查看
日历月视图今日事项摘要卡打开日历时顺带看到

所有入口点击 → 统一跳转 日历日视图改造页（唯一落地页）。

7.6 静默规则
场景 处理
用户已开启小爱简报跳过早间推送（简报已覆盖），仅保留午间+晚间
勿扰模式开启中延后到勿扰结束后立即补推
用户正在通话中延后到通话结束后5分钟
用户刚点击了推送（5分钟内）下一时段正常推，不跳过
用户标记了"不相关"同类事项置信度降低，连续3次后该类不再主动推
连续3天推送均无事项降频为仅早间推送
连续7天无互动暂停推送，下次打开日历时恢复
